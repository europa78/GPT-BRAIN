# Proposal Factory
> One-liner: the digital-marketing proposal is Mike's most-repeated deliverable; it evolved from email-format PDFs to interactive single-file HTML web experiences generated by a reusable master prompt.

## Format evolution
1. **Email-style proposals** (through 2025) — e.g. Rucco/Compass: services recommendation (LSA + PPC + social) wrapped in a pain-first email.
2. **Deep-research phase (2025-09):** formal study of B2B home-services proposal formats — infographics, PDFs, slides, interactive/web (Qwilr/Proposify-class tools evaluated, then bypassed in favor of building from scratch).
3. **Interactive HTML proposals (2026-01):** a **master prompt** converts proposal content (data, links, screenshots) into a single-file HTML/CSS/JS one-pager — sticky header, smooth scroll, reveal-on-scroll, and a **"Strategy Builder"** (toggle services → live scope/price). Reference build: Welter Construction (Elk River, MN roofing).
4. **Refinement lesson:** first version was overloaded → "leaner, fewer options, easier to digest; black/white/tan only." Reduce choice, increase readability.

## Canonical structure (from Welter build)
Executive summary → situation/competitor gap → recommended services (toggleable) → pricing (from the 2026 sell sheet, see [[STB Digital]]) → next steps.

## Rules of thumb
- Proposal = product demo of the agency itself; an interactive site *is* the pitch for web services.
- Price from the standing sell sheet, don't improvise ("STBD Core Services – Sell Sheets w/ Pricing 2026").
- The master prompt lives in the 2026-01-24 thread; treat it as the factory's source code. `confidence: preview-only` (full prompt text truncated)
