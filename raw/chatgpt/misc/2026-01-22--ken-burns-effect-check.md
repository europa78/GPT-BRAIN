**GPT:** That homepage “zooms around / zooms in & out quickly at different speeds” is almost always one of these two patterns:

## 1) A slider doing “pan + zoom” (Ken-Burns–style, but faster/variable)
Even if people call it “Ken Burns,” the modern version is usually a **hero slider** where each slide image gets a different **scale + translate** animation, and the slider changes slides while the animation is running.

**How it’s accomplished**
- The image sits inside a frame with `overflow: hidden;`
- The image itself is animated with `transform: scale(...) translate(...)`
- Speeds vary because each slide has different:
  - `animation-duration` / `transition-duration`
  - easing curve (`ease`, `cubic-bezier(...)`)
  - transform target (bigger zoom, different direction)
- JavaScript (or a plugin) swaps slides by toggling an “active” class.

**Typical CSS structure**
```css
.hero-slide { overflow: hidden; }
.hero-slide img { will-change: transform; transform-origin: center; }

.hero-slide.is-active img {
  transition: transform 6s cubic-bezier(.2,.8,.2,1);
  transform: scale(1.18) translate(-2%, 1%);
}
```

Or with keyframes (more “zooming around” feel):
```css
@keyframes panzoomA {
  0%   { transform: scale(1) translate(0,0); }
  100% { transform: scale(1.25) translate(-3%, 2%); }
}
.slideA img { animation: panzoomA 7s ease-in-out both; }
.slideB img { animation: panzoomA 12s ease-in-out both; } /* different speed */
```

## 2) A parallax/zoom animation layer (not a slider, but “motion background”)
Some themes animate **background images** (or layered images) using:
- animated `background-position`
- `transform: scale()` on pseudo-elements (`::before`)
- scroll-driven parallax + small zoom “pulses”

This is common in page builders (like Beaver Builder) where the animation lives in generated CSS.
