### ✅ **When It’s *Usually* Allowed (Unofficially)**

The FDA generally **uses enforcement discretion** and *may allow* personal importation of medications **if all of the following are true**:

### Scenario: You click on a Google result or type a URL.

1. **DNS Lookup**
   - Your browser asks a **DNS server** to translate the domain (e.g., `grillmaster.com`) into an IP address (e.g., `192.0.2.1`).

2. **Connecting to the Server**
   - Browser connects to the website’s web server via **TCP/IP**.
   - A **TLS/SSL handshake** occurs if the site uses HTTPS (encrypted).

3. **HTTP Request**
   - Your browser sends an **HTTP request** for the web page.
     ```
     GET /best-grills-2025.html HTTP/1.1
     Host: www.grillmaster.com
     ```

4. **Server Response**
   - The web server sends back the HTML, CSS, JS, and media files.
   - These files are cached by browsers or CDNs (like Cloudflare or Akamai).

5. **Rendering the Page**
   - Your browser parses the HTML.
   - Runs any JavaScript and loads styles/images.
   - Displays the final web page for interaction.

> ⏱ Tools like **CDNs**, **browser caching**, and **lazy loading** speed this up significantly.

## 🛠️ PART 1: DEFINE YOUR SERVER'S PURPOSE

Start by answering:  
**What will this server do?**
- 🔹 Host a website?
- 🔹 Run your own cloud (e.g., Nextcloud)?
- 🔹 Store and serve files (media, documents)?
- 🔹 Run a database (e.g., PostgreSQL)?
- 🔹 Be a VPN, email, or game server?

---

## 🧰 PART 2: CHOOSE YOUR HARDWARE

### A. 🧑‍💻 **Option 1: Repurpose a PC**
- A spare desktop or laptop with at least:
  - **2+ CPU cores**
  - **4+ GB RAM** (8+ preferred)
  - **SSD or HDD with 50+ GB**
- Ideal for local learning or small deployments

### B. 🖥️ **Option 2: Build or Buy a Server**
- Custom PC/server with:
  - **Intel/AMD CPU** (Xeon for performance)
  - **8–64 GB ECC RAM** (for databases or multiple users)
  - **SSD/NVMe** for fast I/O
  - **Good power supply & cooling**

### C. ☁️ **Option 3: Use a Virtual Server (VPS)**
- Services like **DigitalOcean**, **Linode**, **Vultr**, or **AWS Lightsail**
- Costs ~$5–$20/month
- Always on, globally accessible, no physical setup

---

## 💽 PART 3: INSTALL AN OPERATING SYSTEM

### A. Best Choices:
| OS        | Best For                       | Notes                                |
|-----------|--------------------------------|--------------------------------------|
| **Ubuntu Server** | General-purpose, easy CLI         | Most popular, good docs              |
| **Debian**        | Stability and minimalism         | Lightweight                          |
| **CentOS/AlmaLinux** | Enterprise-grade services       | Red Hat based                        |
| **Windows Server** | .NET apps or Windows ecosystem | Licensing costs, GUI-heavy           |

➡️ **Download ISO**, flash to USB (using [Rufus](https://rufus.ie)), boot from it, and install.

---

## 🌐 PART 4: CONFIGURE NETWORKING

### A. For Home/Office Servers:
- Set a **static IP** on your LAN (e.g., `192.168.1.100`)
- Configure your **router’s port forwarding** for:
  - Port 80 (HTTP)
  - Port 443 (HTTPS)
  - Other custom services (e.g., 22 for SSH, 21 for FTP)

### B. Public Access:
- Use **dynamic DNS** if your home IP changes (services like No-IP or DuckDNS)
- Or get a **static IP from your ISP**

---

## 🔐 PART 5: SECURE YOUR SERVER

1. **Create a non-root user**
2. Enable **SSH** and use **key-based login**
3. Install a **firewall**:
   - `ufw allow 22`, `ufw allow 80`, `ufw enable`
4. Keep packages updated:
   - `sudo apt update && sudo apt upgrade`
5. Optional: Install **Fail2ban** for brute-force protection

---

## 💻 PART 6: INSTALL SOFTWARE & SERVICES

### A. Web Server
```bash
sudo apt install nginx
```
or:
```bash
sudo apt install apache2
```

### B. Database (Optional)
```bash
sudo apt install mysql-server
```
or:
```bash
sudo apt install postgresql
```

### C. PHP (for dynamic sites)
```bash
sudo apt install php php-fpm php-mysql
```

### D. Let’s Encrypt SSL
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx
```

### E. Self-Hosting Apps (Optional)
- **Nextcloud** – Private cloud
- **Plex** – Media server
- **WordPress** – Website/blog
- **Docker** – Run multiple services with containers

- Use `ping`, `curl`, or `nmap` to test accessibility
- Host public websites, intranet portals, APIs, etc.
